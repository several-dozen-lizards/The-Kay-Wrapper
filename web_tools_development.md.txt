# Web Tools Integration - Complete Build

**Status:** Ready for deployment  
**Date:** 2025-12-17  
**Components:** Real web scraping + tool use framework + curiosity integration

## What We Built

Kay can now **autonomously search the web and fetch pages** during conversations, especially during curiosity sessions. This uses REAL web scraping (not nested API calls), making it cheap and reliable.

### Architecture

```
Kay's conversation
  ↓
Kay's LLM decides to search
  ↓
Tool Use Handler intercepts
  ↓
Real Web Scraping (DuckDuckGo or SerpAPI)
  ↓
Results fed back to Kay's LLM
  ↓
Kay synthesizes information
  ↓
Response to user
```

**KEY ADVANTAGE:** No nested Claude API calls! Web searching uses actual HTTP requests + BeautifulSoup scraping. Much cheaper than calling Claude to call Claude.

## Files Created

## How It Works

### Normal Conversation (No Tools)
```
User: "How are you doing?"
Kay: [normal response, no tools used]
```

### Curiosity Session (Tools Enabled)
```
User: "I want to explore my scratchpad items about AI safety"
Re: [clicks "Start Curiosity Session" button]
Kay: [reads scratchpad items]
Kay: "I see you flagged some things about AI alignment. Let me search for recent developments."
Kay's LLM: [uses web_search("AI alignment recent developments")]
Tool Handler: [executes real DuckDuckGo search]
Tool Handler: [returns results to Kay]
Kay: [synthesizes information]
Kay: "Based on recent research, there are three major approaches..."
```

### Tool Use Flow

1. **Kay decides to use a tool**
   - LLM response includes tool_use block
   - Example: `web_search(query="transformer architecture innovations")`

2. **Tool Handler intercepts**
   - Extracts tool name and parameters
   - Looks up registered function
   - Executes: `web_tools.web_search("transformer architecture innovations")`

3. **Real scraping happens**
   - HTTP request to DuckDuckGo (or SerpAPI)
   - Parse HTML results
   - Extract titles, URLs, snippets
   - Return formatted dict

4. **Results fed back**
   - Tool Handler formats results for API
   - Sends as tool_result to Kay's LLM
   - Kay reads results and continues thinking

5. **Kay synthesizes**
   - Kay's LLM processes search results
   - Generates response incorporating new information
   - Returns final text to user

## Available Tools

### Web Tools (Always Available in Curiosity Mode)

**`web_search(query: str)`**
- Searches the web for current information
- Returns: List of results with titles, URLs, snippets
- Example: `web_search("latest Claude API updates")`

**`web_fetch(url: str)`**
- Fetches and extracts text from a webpage
- Returns: Clean text content (first 8000 chars)
- Example: `web_fetch("https://anthropic.com/news")`

### Curiosity Tools (Available in Curiosity Sessions)

**`store_insight(content: str, category: str)`**
- Stores findings in autonomous memory
- Example: `store_insight("Transformers use attention mechanism", "architecture")`

**`mark_item_explored(item_id: str, summary: str)`**
- Marks scratchpad item as explored
- Example: `mark_item_explored("item_123", "Found that attention is key to performance")`

## Troubleshooting

### "Module not found: requests"
```bash
pip install requests beautifulsoup4 lxml
```

### "Tool 'web_search' not registered"
Tool handler wasn't initialized. Check Step 2 of integration guide - tools need to be registered at startup.

### "DuckDuckGo search failed"
DDG can be flaky. Options:
1. Retry the search
2. Use SerpAPI instead (add API key to .env)
3. Check internet connection

### "Tools not available during conversation"
Tools are only enabled during curiosity sessions by default. Check that:
1. Curiosity session is active
2. `enable_tools=True` passed to `get_llm_response()`
3. Tool handler is initialized

### UI buttons not showing
Old UI cached. Solutions:
1. Restart Kay UI completely
2. Check that `autonomous_ui.py` has the buttons (lines 156-181)
3. Check that `autonomous_ui_integration.py` wires them (lines 289-290)

## Performance Notes

### Search Speed
- **DuckDuckGo:** ~1-3 seconds per search
- **SerpAPI:** ~0.5-1 second per search
- **Page fetch:** ~1-2 seconds per page

### API Costs
- **Web search:** $0 (using DuckDuckGo) or ~$0.002 per search (SerpAPI)
- **Web fetch:** $0 (just HTTP requests)
- **Kay's LLM call:** Normal Claude API rates (~$0.015 per call with caching)

**Total cost per curiosity session:** ~$0.02-0.05 (mostly Kay's LLM, not the web tools)

Compare to nested API approach: Would be ~$0.30+ per session (10x more expensive!)
